Runner: AI agent (Claude Code session).

This log captures the **post-Codex-Round-1 (restart) state**. Earlier in-session runs surfaced two pre-existing bugs and one critical methodology defect that the Codex restart caught:

1. **Rule-ordering shadow bug** (image-class): the broader same-origin `pages` rule sat BEFORE the image-class rules in the `runtimeCaching` array, so workbox matched `/_next/image?url=...` against the page rule first and the `next-image` cache stayed empty. Phase 5 reorders so image-class rules precede the page rule.
2. **Rule-ordering shadow bug** (static-assets + next-data): the same shadow pattern affected the `static-assets` SWR rule and the `next-data` NetworkFirst rule. Same-origin `.js`/`.css` were being handled as NetworkFirst pages instead of SWR static payloads, and `/_next/data/*.json` would never have reached its own rule. Phase 5 reorders these too.
3. **Lighthouse-was-measuring-login methodology defect**: every prior in-session Lighthouse run hit `/login` due to middleware redirect — confirmed via the `finalDisplayedUrl` field in the LH JSON. The run log's pre-restart no-regression claims were invalid (measuring `/login` four times with different `requestedUrl`). Recipe now extracts the seed user's `next-auth.session-token` cookie via `/api/login` and passes it through `--extra-headers`.

Notes:

- **Sub-check 2b is a two-layer assertion** (post-R3-finding remediation). The initial `<img>` + `naturalWidth > 0` design was too loose — it proved the browser ended up with a usable image but not WHO served it (Codex R3 Probe A). The first remediation attempt used `response.fromServiceWorker()` via a Playwright listener, but the browser's HTTP/memory cache short-circuited the second-pass `<img>` loads before they reached the SW, so no response event fired. Final design: for each cached URL, (a) `caches.match(url, { ignoreVary: true })` against the workbox image caches directly — proves the SW has the URL cached (workbox stores image responses with `Vary` headers, so the `ignoreVary` flag is necessary, matching workbox's own runtime behavior); (b) `<img>` decode under offline — proves the user-perceived outcome. The two together prove both the contract (SW cache hit) and the experience (image renders).
- **Step 3 now asserts real `/schedule` body content** (Codex Round 1 finding D): `fromServiceWorker === true && status === 200` alone is insufficient — could be a workbox-precache fallback or login redirect. The offline reload's `finalUrl` must end with `/schedule` (no redirect), the body must reference `/_next/static/chunks/` (authed-route shell), and must NOT contain login-form markers (`type="email"`, "Sign in"). All three assertions pass on the current build.
- **Step 3 verifies offline fallback, not the literal PRD AC #3 timing band of "navigation under 6 s with 10 s latency".** Playwright's CDP `Network.emulateNetworkConditions` latency does not propagate to service-worker-initiated fetches on the Chromium version bundled with Playwright 1.61.1 — verified empirically: under 10 s applied latency, the SW's network fetch resolved in ~20 ms, far below any timeout boundary. The 5 s timeout VALUE is verified at the source level by Step 1's block-targeted grep (`cacheName: 'pages'` → `networkTimeoutSeconds: 5`).
- **Step 4 framing clarification (vs literal PRD AC #4 "seed-data mutation + reload"):** workbox's page rule caches the page DOCUMENT (HTML shell), not the per-route data. In the App Router, page documents are client-rendered shells that mount React components which fetch via React Query against `/api/data/*` — and `/api/*` has no SW rule (data flows through React Query's own caching layer, not workbox). Mutating seed data + reloading therefore tests React Query's `staleTime` semantics, not Phase 5's page-rule change. Codex Round 1 Probe B traced this through `apps/attendee/app/(authenticated)/(app)/schedule/page.tsx` and confirmed: the page passes empty props and fetches client-side via `useScheduleData()`, no server-rendered seed data in the page document. The cache-poison test is the correct contract test for what Phase 5 actually changes.
